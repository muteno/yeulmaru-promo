## 2. 핵심 문제

1. **공유 ID 부재** — 같은 공연이 3개 시트에 각기 다른 표기로 존재. 공연마스터만 ID(예 `260514_01`) 보유, 나머지는 자유텍스트 공연명.
2. **조인 매칭률 낮음** — 일일입력 34공연 중 공연마스터 매칭 7(21%). 나머지 27개는 일별 판매는 있으나 기준석/목표/수익성 미상 → 판매현황 분석에서 누락.
3. **명칭 표기 불일치** — 꺾쇠 혼용(`<>` vs `〈〉`), `_` 구분자, `- 여수` 접미사, 불규칙 공백. → 정규화로 대부분 해소 가능.
4. **개념 이중저장** — 수익성이 공연마스터엔 컬럼, 운영대장엔 없음(런타임 규칙). 단일 원천 위배.
5. **운영대장 결손** — 2025 브런치 하위에디션·위크·플라멩코·Pre Festival·2025 신년음악회 등 미입력. 2026 신규는 공연마스터에만 존재.

## 4. 단계별 정제 계획 (제안)

### Phase 1 — 조인 로직 강화 (코드, 안전·가역, git 복원 가능)
- 공유 정규화 함수 `_normName()`: `- 여수`/꺾쇠/언더스코어/공백 제거, 연도 인지.
- `_salesBuild`: 공연마스터에 없는 일일입력 공연은 **운영대장 폴백**(기준석/장르/구분) + `_profitType`(수익성)으로 보강 → 27개 누락 공연 분석 편입.
- SharePoint 데이터 변경 없음. 즉시 테스트 가능.

### Phase 2 — 데이터 결손 보충 (시트 쓰기, 확인 후)
- 운영대장에 `수익성`·`목표점유율` 컬럼 추가(or `운영_수익성` override 시트).
- 진짜 누락 공연 입력: 브런치 하위에디션 4건, 위크, 플라멩코, Pre Festival, 2025 신년음악회 등.

### Phase 3 — 공연ID 단일 차원 (구조 개편)
- 운영대장을 척추로 `공연ID`(PK) + `표준공연명` + `별칭(|구분)` 컬럼 신설.
- 일일입력·신청내역·회차상세가 공연ID(FK) 참조. 향후 CRM 고객도 공연ID로 결합.

## 5. 결정 필요 사항
1. ID 체계: 공연마스터식 `YYMMDD_NN`(시작일 기반) vs 신규 일련번호?
2. 단일 원천 위치: 운영대장 확장 vs 별도 `운영_공연사전` 신설?
3. 시리즈 그룹 공식화 여부(브런치/신년/실내악/위크/헬로오페라 — 연도별 비교용)?
4. 재연(시카고 등) 처리: 작품ID와 공연(에디션)ID 분리?
