# 전시 DB 설계 (공연 DB 미러) — 260620

> 전시도 공연과 **동일 수준**으로 분석 가능하게, 공연 DB 구조를 그대로 미러했다.
> 공연 시트(`운영_일일입력`/`운영_공연마스터`/`운영_공연색인`)는 **무변경** — 전시는 전용 시트로만.
> 원본: 2024·2025·2026 일별 엑셀 3개(각 일자=시트, 블록=전시). 파싱·검증·집계는 `docs/build_db`(스크립트).

## 1. 시트 구성

| 전시 시트 | 대응 공연 시트 | 역할 |
|---|---|---|
| **운영_전시일일** | 운영_일일입력 | 일자×전시 **사실(fact)** — 일일 증감 + 누계 |
| **운영_전시마스터** | 운영_공연마스터 | 전시 1행 **차원(dimension)** — 목표·최종·수익성·상태 |
| *(전시ID = 색인 역할)* | 운영_공연색인 | 별도 색인 생략 — `전시ID`가 조인키 겸 식별자 |

조인키 **`전시ID = YYMMDD_NN`** (시작일 + 동일시작일 순번) — 공연ID 규약과 동일.

## 2. 운영_전시일일 (937행 · 2024+2025+2026)
`기준일자(YYYYMMDD) · 전시ID · 전시명 · 일일유료 · 일일무료 · 일일총인원 · 일일금액 · 누계유료 · 누계무료 · 누계총인원 · 누계금액 · 점유율`

- **일일\* = 그날 하루치**(전일 누계 대비 증감). 첫 기록일은 그날 누계 = 일일. → *"각각 따로"* 분석(일별 페이스·스파이크) 가능.
- **누계\* = 개막~해당일 누적.** 공연 `합계좌석`과 동일 성격.
- 검증: 전 전시에서 **Σ(일일유료) = 최종 누계유료** (증감 정확), 누계 단조증가.

## 3. 운영_전시마스터 (25개 전시)
`전시ID · 전시명 · 연도 · 시작일 · 종료일 · 운영일수 · 목표관객 · 목표금액 · 최종유료 · 최종무료 · 최종총인원 · 최종매출 · 최종점유율 · 수익성 · 상태`

- **목표관객 = 점유율의 분모(역산).** 원본 `점유율 = 누계유료 / 목표관객`이라 점유율이 100% 초과 가능(예: 앙리 마티스 182%, 이존립 430%). 즉 **점유율 = 목표 달성률**(좌석 점유가 아님). 예술성 전시는 목표관객을 낮게 잡아 초과달성이 흔함 = 정상(도달·다양성 지표).
- **수익성(공공/상업/무료)** = 규칙 휴리스틱 **초안** — 시트에서 직접 수정 가능(공연의 `운영_수익성` override와 동일 철학). 가족·캐릭터·블록버스터=상업, 창작/입주/지역작가=공공, 위크/오픈스튜디오=무료.
- **상태** = 종료일 vs 오늘 → 종료/진행중/예정.

## 4. 파싱 신뢰성 (왜 믿어도 되나)
- **블록 단위 추출** — 각 전시 블록의 자기 `구분`·`누계 총 금액` 앵커에서만 값을 읽어 다중전시일 **점유율 혼입 0**.
- **상대 컬럼** — 2024/2025는 2026 대비 한 칸 우측 시프트. 앵커(`전시명 :`/`구분`/`누계 총 금액`) 기준 오프셋으로 3개년 동일 처리.
- **제목 정규화** — 같은 전시의 표기 흔들림(공백/꺾쇠) 통합(예: 2025 `어린이 미술전<냠냠>`=`어린이 미술전 <냠냠>`).
- **팬텀 블록**(제목 `"4"` 등) 제외, **무료 관람 프로그램**(오픈스튜디오) 금액 0·점유율 없음으로 정규화.
- 앙리 마티스 등 일부 원본 `총인원` 필드가 유료만 담는 오기 → **총인원 = 유료+무료**로 일관 계산.

## 5. 반영
```
DB_PW=<슈퍼비번> node docs/260620_exhib_db_ingest.mjs           # 미리보기
DB_PW=<슈퍼비번> node docs/260620_exhib_db_ingest.mjs --write   # 운영_전시일일 + 운영_전시마스터 풀라이트
```
공연 시트 무영향. 기존 2026-only `운영_전시일일`은 이 풀라이트로 3개년·신스키마(전시ID·일일증감 포함)로 교체됨.
